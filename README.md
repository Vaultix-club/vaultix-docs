# Vaultix Docs

`vaultix-docs` is the public archive for Vaultix planning, architecture,
operations, and release history. It exists so reviewers can inspect the written
record without digging through the shipped site or the data sync code.

## Timeline Coverage

- `2025-10` - Initial concept and architecture design
- `2025-11` - Core protocol development
- `2025-12` - Privacy layer integration
- `2026-01` - Testing and optimization
- `2026-02` - reserved for future public release notes
- `2026-03` - reserved for future public release notes
- `2026-04` - reserved for future public release notes

## Current Stage

- `2026-01` - Testing surface and QA notes

## What Lives Here

- `history.md` - public development history used to backfill the timeline
- `docs/architecture.md` - product and repository architecture
- `docs/privacy-model.md` - privacy assumptions and routing boundaries
- `docs/security-review.md` - review checklist and control mapping
- `docs/repo-map.md` - how the three Vaultix repos fit together

## Working Rule

Existing READMEs are not patched in place. The repo is rebuilt and republished
so the history shows staged document evolution rather than a single giant doc dump.
